# Contributing

## Requirements

- Every non-trivial change must reference a FIP.
- Feature changes require linked `Accepted` or `Scheduled` proposal.
- External toolchain/compiler/assembler/linker use is prohibited in normal build paths.

## Pull Request Checklist

1. Linked FIP ID.
2. Updated status or rationale.
3. Tests for behavior changes.
4. Reproducibility impact noted.
5. Compatibility impact noted.

## Review Focus

- Safety and behavior correctness.
- Determinism and reproducibility.
- Alignment with full-independent constraints.
